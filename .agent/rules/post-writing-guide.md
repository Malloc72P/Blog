# 블로그 포스트 작성 가이드

이 문서는 블로그 포스트 작성 시 일관성을 유지하고 품질을 높이기 위한 지침입니다.

## 1. Frontmatter 설정

모든 포스트는 MDX 파일 상단에 `frontmatter`를 정의해야 합니다. 이를 위해 `@libs/frontmatter`의 유틸리티 함수를 사용합니다.

### Import 및 사용법
```tsx
import { frontmatter } from '@libs/frontmatter';

export const metadata = frontmatter({
  title: '포스트 제목',
  description: '포스트에 대한 짧은 요약 (1~2문장)',
  seriesId: '카테고리명 (예: frontend, ai)',
  postId: '포스트-식별자 (파일명과 동일해야 함)',
  tags: ['태그1', '태그2'],
  date: 'YYYY-MM-DD HH:mm',
});
```

### 주의 사항
- `postId`는 파일 시스템 경로로 사용되므로 파일 이름과 정확히 일치해야 합니다.
- `date`는 `YYYY-MM-DD HH:mm` 형식을 준수해야 합니다.

## 2. 글 작성 패턴

### 헤딩 구조
- `#` (H1)은 포스트 내에서 한 번만 사용하며, 주로 큰 섹션의 시작으로 활용합니다. (제목은 metadata에 정의된 값을 시스템이 사용함)
- `##` (H2), `###` (H3)를 사용하여 논리적인 구조를 잡습니다.

### 코드 블록
- 언어 명시를 원칙으로 합니다. (예: ` ```typescript `, ` ```json `)
- 필요한 경우 파일 경로를 주석으로 포함합니다.

### 이미지 및 미디어
- 이미지는 public 폴더에 위치시키고, 절대 경로로 참조합니다. (예: `/image-name.png`)
- `alt` 텍스트를 반드시 작성하여 접근성을 높입니다.

## 3. 작문 가이드

### 서술 관점 및 구조

- **경험 중심 서술 (1인칭)**: 3인칭의 딱딱한 설명문보다는 작성자의 실제 경험과 노하우가 드러나는 1인칭 관점(`나는...`, `저의 경우...`)을 사용합니다. 이는 독자에게 실무적인 신뢰감을 줍니다.
- **메커니즘 중심 설명**: 단순히 개념을 정의하는 데 그치지 않고, 구체적인 코드 예시나 설정 값(예: `trigger`, `model_decision` 등)을 통해 기술이 실제로 어떻게 작동하는지(Mechanism)를 상세히 다룹니다.
- **친숙한 비유 사용**: 추상적이고 거창한 표현 대신, 독자가 이미 알고 있는 개념(예: "에이전트 기능이 강화된 VS Code")을 비유로 사용하여 진입 장벽을 낮춥니다.

### 어조 및 표현 (담백한 표현 유지)

- **과도한 수식어 지양**: 감정적이거나 과장된 표현("최고의", "압도적인", "혁명적인")을 피하고, 객관적이고 담백한 사실 위주로 기술합니다.
- **설명이 거창해지지 않도록 주의**: 어려운 용어를 나열하기보다 친근하고 이해하기 쉬운 문장을 사용합니다. (예: "설명이 좀 거창했는데, 쉽게 생각하면..." 등)
- **권장 표현 사례**:
  - 피해야 하는 표현: "가장 강력한", "압도적인", "비교할 수 없는", "하늘과 땅 차이"
  - 권장하는 표현: "더 나은 방법은...", "효율적인 방법으로...", "실제로 사용해 보니..."
