# RealGrid 블로그 작성 스킬

RealGrid 기술 블로그의 고유한 작문 패턴을 재현하는 스킬이다.
아래 패턴을 철저히 따라 포스트를 작성한다.

---

## 작문 원칙

### 1. 제목 패턴
두 가지 형태 중 하나를 선택한다:
- **이유 제시형**: `[개념 A] vs [개념 B]: 왜 [B] 대신 [A]인가?`
- **범위 제시형**: `[주제]: [시작점]부터 [끝점]까지`

제목에서 "이 글을 읽으면 무엇을 얻는지"를 명확히 약속한다.

### 2. 글의 흐름 (필수)
반드시 아래 순서를 지킨다:

```
왜(배경/문제) → 무엇(개념/대안) → 어떻게(실전/설정) → 마무리
```

- **왜**: 독자가 이 글을 읽어야 하는 이유. 문제 상황이나 기술적 배경.
- **무엇**: 핵심 개념 소개. 둘 이상의 선택지가 있다면 비교표/불릿으로 나란히 제시.
- **어떻게**: 실전 예제 또는 단계별 설정. 코드 블록 포함.
- **마무리**: 본문 핵심을 1~2문장으로 압축 + 독자에게 적용을 권유.

### 3. 중간 정리 (선택, 글이 길 경우 권장)
개념 설명 후 실전 예제 전에 "중간 정리" 또는 "핵심 요약" 섹션을 넣어 독자가 길을 잃지 않게 한다.

### 4. 문장 스타일
- 문장은 짧고 단호하게. 한 문장에 하나의 생각만.
- `따라서`, `반면`, `즉` 같은 대비·인과 접속사로 논리를 명확히 끊는다.
- 어조: 존댓말이되 딱딱하지 않게. `~해주시면 됩니다`, `~하셔도 됩니다` 사용.
- 감탄사, 불필요한 서론 없이 바로 본론.

### 5. 코드 블록 규칙
코드 블록은 반드시 앞뒤 설명 문장으로 감싼다 (샌드위치 구조).

```
[이 코드가 왜 필요한지 설명]

\`\`\`언어
코드
\`\`\`

[코드의 핵심 포인트 불릿 설명]
```

- 출력 결과가 있다면 주석으로 함께 보여준다.
- 독자가 모를 수 있는 세부사항(파일명, 빈칸 여부 등)은 친절하게 명시한다.

### 6. 비교 구조
두 선택지를 비교할 때는 나란히 제시 후 **추천 이유**를 별도 섹션으로 설명한다.
"A가 더 낫다"는 결론을 먼저 내리지 않고, 비교 → 이유 → 결론 순서를 지킨다.

### 7. 마무리 섹션
- "마치며" 또는 "마무리하며" 제목 사용.
- 본문 핵심을 1~2문장으로 압축.
- 독자에게 실제 프로젝트 적용을 권유하는 문장으로 끝.
- 참고한 공식 문서가 있다면 하단에 출처 링크 명시.

## 글의 깊이 기준

| 글 성격 | 깊이 | 예시 |
|---|---|---|
| 실용 가이드 | 얕고 빠르게 | 설정 방법, 마이그레이션 |
| 개념 튜토리얼 | 깊고 천천히 | 동작 원리, 실전 예제 2개 이상 |

- 실용 가이드: 단계별 설정 위주, 이유 설명 간결하게.
- 개념 튜토리얼: 기본 구조 → 동작 원리 → 중간 정리 → 실전 예제 순서 필수.

## 체크리스트 (작성 후 확인)

- [ ] 제목이 "이유 제시형" 또는 "범위 제시형"인가?
- [ ] 왜 → 무엇 → 어떻게 → 마무리 순서를 지켰는가?
- [ ] 코드 블록 앞뒤에 설명 문장이 있는가?
- [ ] 불필요한 감탄사나 긴 서론이 없는가?
- [ ] 마무리에서 핵심 압축 + 적용 권유를 했는가?
- [ ] 두 선택지 비교 시 비교 → 이유 → 결론 순서인가?

## 출력 규칙

- 작성한 포스트는 **기존 파일을 수정하지 않는다**.
- 반드시 `tmp/` 폴더에 임시 파일로 저장한다.
  - 경로 형식: `tmp/[포스트-식별자]-realgrid-blog.md`
  - 예: `tmp/antigravity-realgrid-blog.md`
- 임시 파일 저장 후 사용자에게 경로를 안내한다.
